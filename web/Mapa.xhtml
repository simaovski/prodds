SOU ADMINISTRADOR SOU COLABORADOR

ADMINISTRADOR

O Administrador é de fundamental importância para o gerencimento de produção no ambiente DDS. Desta forma, ele é o profissional responsável por cadastrar um novo projeto no ProDDS e cadastrar o ciclo de vida utilizado por este projeto. Além disso, é de sua responsabilidade selecionar quais colaboradores serão Gerentes de Equipe.

Responsabilidades

São suas responsabilidades no sistema:
1) Cadastrar um Ciclo de Vida
2) Cadastrar um Projeto
3) Selecionar Gerente de Equipe
4) Gerenciar o Status do Projeto (Concluído/Cancelado/Desenvolvimento)

PERMISSÕES DE ADMINISTRADOR

1) Cadastrar Ciclo de Vida

O Ciclo de Vida representa o conjunto de fases e atividades desenvolvidas em um projeto. Em geral, os Ciclos de Vida possuem a caracteristica de ser simples e direto. No ProDDS você pode personalizar o seu Ciclo de Vida a partir do cadastro de fases e atividades.

Passo a Passo: Página Inicial ADM > Selecionar opção Cadastrar Processo

2) Cadastrar Projeto

O Projeto representa o sistema a ser desenvolvido. Desta forma, para ocorrer seu cadastro são necessárias as informações sobre o Nome do Projeto e uma breve descrição do mesmo. Para resultados efetivos de gerenciamento também é necessário que o Administrador selecione a data de início e qual o Ciclo de vida Utilizado.

Passo a Passo: Página Inicial ADM > Selecionar opção Cadastrar Projeto

3) Selecionar Gerentes de Equipe

Cada projeto poderá possuir N Gerentes de Equipe. O Gerente de Equipe é responsável por gerenciar atividades e artefatos da sua equipe. Desta forma, para cadastrar um novo Gerente no Projeto, o usuário colaborador deverá enviar uma solicitação de Gerência ao Administrador do Projeto. O passo a passo dessa ação é demostrado na seção Solicitar Gerencia.

Passo a Passo: Página Inicial ADM > Notificações

4) Status do Projeto

O Status do projeto pode possuir três estados:
Desenvolvimento: O projeto e suas atividades estão em progressão
Concluído: Neste estado o projeto já foi concluído e nenhuma Atividade deste pode ser desenvolvida
Cancelado: Um projeto cancelado significa que suas atividades foram pausadas. O Administrador pode retornar o Status de um Projeto Cancelado para Em Desenvolvimento, porém, será criado uma nova instância do Projeto.

Passo a Passo: Página Inicial ADM > Selecionar opção Cadastrar Projeto

COLABORADOR

Os Colaboradores de um projeto DDS são os responsáveis pelo desenvolvimento de atividades e construção de artefatos durante a construção de um sistema. No ProDDS, um Colaborador pode desenvolver uma atividade designada, construindo uma base de meta dados sobre horários e atividades.

Responsabilidade

São responsabilidades no sistema:
1) Desenvolver Atividade Designada
2) Enviar Artefato da Atividade

Potencialidade

Um Colaborador pode
3) Gerente de uma Equipe

PERMISSÕES DO COLABORADOR

1) Desenvolver atividade designada

O Colaborador poderá desenvolver uma atividade previamente cadastrada para ele. Durante seu desenvolvimento será salvo o tempo gasto, a atividade desenvolvida e se essa atividade gerou algum artefato. Essas informações serão base para a criação dos meta-dados sobre a produção no ambiente DDS.

Passo a Passo: Página Inicial Colaborador > Selecionar Projeto

2) Enviar artefato da Atividade

O Colaborador pode enviar o artefato oriundo de uma atividade para a supervisão dos Gerentes de Equipe. Todos os Gerentes do Projeto terão acesso à esse artefato, a fim de acompanhar o desenvolvimento do projeto em equipes distintas a sua.

Passo a Passo: Página Inicial ADM > Enviar Artefato

3) Gerente de uma Equipe

O Colaborador também pode solicitar Gerência de um Projeto, esta solicitação será enviada ao Administrador do projeto para Confirmação. Entretando, um colaborador apenas possui permissão de solicitar gerência quando ele não esta alocado como Gerente no mesmo projeto.

Passo a Passo: Página Inicial ADM > Selecionar opção Cadastrar Projeto

Gerente de Equipe

O Gerente de uma Equipe é responsável por gerenciar todo o desenvolvimento de um projeto, realizando e verificando as competências de todos os colaborados que participam de sua equipe. Este profissional tem um papel fundamental na gerência de produção...

Responsabilidade

É responsabilidade no sistema:
1) Cadastrar Equipe
2) Cadastrar Atividade
3) Designar Atividade para Colaborador da Equipe
4) Selecionar Status da Atividade
5) Gerar relatórios de sua Equipe

Permissões do Gerente de Equipe

1) Cadastrar Equipe

Quando a solicitação do Colaborador é confirmada pelo Administrador do projeto, este passa a ser Gerente. Como gerente, uma de suas tarefas é realizar o cadastro de sua equipe. Para isso será necessário cadastrar o nome e a descrição da Equipe. Devido a pluralidade do DDS é necessário que o gerente também cadastre a localização de sua equipe. Para realziar o cadastros dos Colaboradores da sua equipe, o Gerente pode optar por:
1) Enviar solicitação aos Colabodores cadastrados no ProDDS ou;
2) Confirmar a solicitação de cada colaborador. Desta forma, cada Colaborador que deseje integrar a Equipe deverá enviar uma solicitação.

Passo a Passo: Página Inicial Gerente > Cadastrar Equipe

2) Cadastrar Atividade

Um gerente pode optar por cadastrar uma Atividade ou utilizar as atividades já previamente cadastradas pelo Administrador do Projeto. Quando um Gerente opta por cadastrar uma atividade, essa deverá pertencer a uma fase do Ciclo de Vida que o projeto utiliza.

Passo a Passo: Página Inicial Gerente > Cadastrar Atividade

3) Designar Atividade para Colaborador da Equipe

Um gerente deve designar atividades para todos os membros da sua equipe. Desta forma ele pode escolher por selecionar as atividades pertencentes ao projeto (atividades cadastradas pelo Administrador ou por ele mesmo) e selecionar qual Colaborador da Equipe será responsável pelo seu desenvolvimento.

Passo a Passo: Página Inicial Gerente > Cadastrar Atividade Por Usuário

4) Selecionar Status da Atividade

O Gerente pode modificar o Status de uma Atividade que um Colaborador vem desenvolvendo. Sendo assim, os estados possíveis são:
1) Desenvolvimento: A Atividade pode ser desenvolvida pelo usuário
2) Concluído: A Atividade não poderá ser desenvolvida pelo usuário, pois a mesma está completa.

Passo a Passo: Página Inicial Gerente > Cadastrar Status de Atividade

5) Gerar relatórios de sua Equipe

O Gerente também pode, no âmbito de seus deveres, acompanhar o desenvolvimento do Projeto a partir da consulta de todos os relatórios sobre os Log's individuais da sua Equipe. Em cada Log é constato a Colaborador, a Atividade, a Hora e a Data de início e término

Passo a Passo: Página Inicial Gerente > Consultar Relatório

SOBRE O ProDDS

O ProDDS é resultado de um trabalho de conclusão de curso. O foco de pesquisa foi identificado a partir de uma pesquisa de iniciação cientifíca concentrada neste tema. A partir de seu desenvolvimento foi procurado oferecer uma solução de sistema capaz de auxiliar o gerenciamento de produção em ambientes DDS.
